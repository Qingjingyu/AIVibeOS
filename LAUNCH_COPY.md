# Launch Copy

## 1. GitHub Repository Description

A reusable AI development operating system with commands, skills, adapters, QA gates, and release workflows.

## 2. Short Launch Post

我们把过去一段时间里沉淀的 AI 开发流程、skills、commands、工具适配和治理能力，整理成了一个独立可分发的系统：`AIVibe OS`。

它不是一个单独的 Agent，而是一套让 AI 开发从“更快写 Demo”走向“可验证、可回滚、可出货”的开发操作系统。

当前版本：`v1.0.0-beta.1`

适合想把 Claude、Cursor、Codex 等工具接到同一套稳定工作流上的团队。

## 3. Medium Launch Post

我们正式把 `AIVibe OS` 放出来了。

这不是一个新的 AI 编程模型，也不是某个单独的代码助手，而是一套面向真实产品交付的人机协同开发操作系统。

它解决的问题不是“让 AI 写得更快一点”，而是：

- 怎么定义需求
- 怎么拆任务
- 怎么做增量实现
- 怎么留下测试证据
- 怎么做 review / release / rollback

当前版本已经包含：

- 7 个标准 commands
- 25 个 root skills
- Claude / Cursor 适配层
- Claude 项目级 skills 与 subagents
- QA / 发布 / 回滚模板
- 治理脚本和固定 closeout 输出

版本：`v1.0.0-beta.1`

如果你想把 AI 开发从“会写代码”推进到“稳定出货”，AIVibe OS 值得你试一轮。

## 4. Long Launch Post

过去一段时间，我们一直在做一件事：

不是让 AI 再多写一点代码，而是把 AI 开发真正变成一套稳定的交付系统。

于是就有了 `AIVibe OS`。

它的定位很明确：

不是一个单独的万能 Agent，  
不是一堆零散 prompt，  
也不是一个只能演示的 demo 框架。

它是一套面向真实产品交付的人机协同开发操作系统。

当前版本里，已经包括：

- 标准开发阶段命令
- 一整套 root skills
- Claude / Cursor 适配层
- Claude 项目级 skills 与专家会诊 subagents
- Figma 高保真还原能力
- QA / 发布 / 回滚模板
- 治理脚本和固定产物

它想解决的，不是“AI 会不会写代码”，而是：

- 需求怎么定
- 任务怎么拆
- 实现怎么做
- 验证怎么留证
- 发布怎么可控
- 回滚怎么提前准备

当前版本是 `v1.0.0-beta.1`。

它已经适合：

- 团队内部试用
- 对外分发
- 作为项目试点产线

但它还不是终点。

接下来真正关键的，不是继续堆文档，而是拿真实项目去跑，把这套系统从“很完整的开发系统”继续打磨成“真正稳定的自动化出货系统”。
