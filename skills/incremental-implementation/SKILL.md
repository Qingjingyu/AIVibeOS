# incremental-implementation

## 解决什么问题

按薄切片增量实现，减少一次性大改动带来的返工和失控。

## 什么时候用

- 进入实现阶段
- 任务已经拆好
- 需要持续验证

## 不什么时候用

- Spec 还没完成
- 现在是纯分析阶段

## 输入

- Tasks
- 当前代码上下文
- 验收标准

## 输出

- 一次可运行的小改动
- 对应验证结果

## 执行步骤

1. 一次只做一个薄切片。
2. 先改最小必要代码，不顺手扩边界。
3. 每步改完就跑最小验证。
4. 如果失败，先缩小问题，不继续堆改动。
5. 每个切片都保留可回退空间。

## 常见偷懒借口

- “顺手把旁边也一起改了”
- “先全部改完再统一测”
- “这个地方以后再整理”

## 红线与危险信号

- 单次改动跨太多文件和层
- 没有中间验证
- 本轮目标已经漂移

## 验证方式

- 每次改动能单独解释
- 每步都有运行或阅读证据
- 回退成本低

## 交接产物

- 本步改动摘要
- 验证结果
- 下一步建议

## 真实例子

- 页面模块逐步落地
- API 接口按用例增量接入
