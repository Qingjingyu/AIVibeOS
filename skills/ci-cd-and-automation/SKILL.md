# ci-cd-and-automation

## 解决什么问题

把发布前的关键检查尽量自动化，减少“人工应该做但忘了做”。

## 什么时候用

- 准备接入或调整流水线
- 发布前检查
- 把高频手工动作转成自动化

## 不什么时候用

- 还没明确哪些检查值得自动化
- 当前只是临时试验，不需要长期沉淀

## 输入

- 当前交付流程
- 关键检查项
- 失败成本

## 输出

- 自动化建议
- 流水线步骤
- 失败反馈点

## 执行步骤

1. 先找高频、高风险、常漏的动作。
2. 优先自动化最关键的检查，不追求一口气做全。
3. 明确失败时怎么反馈、谁处理。
4. 把手工和自动边界说清楚。
5. 自动化上线后留结果入口。

## 常见偷懒借口

- “先手工跑着”
- “这一步以后再自动化”
- “流水线太复杂，先不管”

## 红线与危险信号

- 高风险动作长期只靠手工
- 自动化失败没有反馈闭环
- 只有流程，没有结果产物

## 验证方式

- 自动化项和手工项边界明确
- 失败反馈路径清楚
- 结果有固定出口

## 交接产物

- 流水线建议或更新说明
- 自动化覆盖范围
- 剩余手工项

## 真实例子

- 发布前检查脚本化
- 固定收尾动作串联
