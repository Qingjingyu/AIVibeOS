# figma-to-code

## 解决什么问题

把 Figma 页面高保真地还原成可运行代码，并保留可验收证据。

## 什么时候用

- 用户提供 Figma URL / node id
- 要求高保真还原
- 不希望 AI 自己重设计

## 不什么时候用

- 只有低保真需求
- 用户明确要重设计
- 没有 Figma 上游真值

## 输入

- Figma 链接或节点
- 页面目标
- 验收要求

## 输出

- 页面实现
- 对稿截图
- 差异说明

## 执行步骤

1. 先走 Figma 真值获取，不先猜。
2. 判断是否为复杂页。
3. 先搭结构，再接素材，再收参数。
4. 在浏览器里留截图证据。
5. 只声明有证据支撑的完成度。

## 常见偷懒借口

- “先照截图抄个差不多”
- “素材后面再换”
- “像素级先口头说一下”

## 红线与危险信号

- 没拿设计上下文就开写
- 用整页截图糊背景交差
- 没有浏览器对稿证据

## 验证方式

- 调用 [../../Top能力/Figmatocode/README.md](../../Top能力/Figmatocode/README.md) 的流程
- 页面截图和设计截图可对照
- 差异有记录
