# verification-before-completion

## 解决什么问题

防止“感觉做完了”就宣称完成，要求在结束前用证据证明。

## 什么时候用

- 准备说“完成了”
- 准备交付或合并
- 准备把结果回报给人

## 不什么时候用

- 还在探索中
- 明确是中间进度同步

## 输入

- 当前改动
- 当前阶段
- 已有验证结果

## 输出

- 完成证据
- 未完成项
- 风险说明

## 执行步骤

1. 先问现在属于哪个阶段。
2. 对照该阶段最小证据要求。
3. 没有证据的地方不能说“已完成”。
4. 区分已完成、部分完成、未验证。
5. 交付前补最关键缺口，而不是包装语言。

## 常见偷懒借口

- “应该没问题”
- “我看过了，大概对”
- “先说完成，后面再补验证”

## 红线与危险信号

- 没跑测试就宣称通过
- 没看页面就说 UI 完成
- 没回滚就说可发

## 验证方式

- 阶段证据齐全
- 结论和证据一一对应
- 未完成项被明确列出

## 交接产物

- 验证摘要
- 证据列表
- 未完成和风险

## 真实例子

- 合并前自检
- 向用户汇报前的完成校验
