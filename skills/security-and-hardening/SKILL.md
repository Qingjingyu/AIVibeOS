# security-and-hardening

## 解决什么问题

在合并或上线前，把明显的安全风险和默认不安全做法压下来。

## 什么时候用

- 处理用户输入
- 新增接口
- 接入认证、权限、密钥、文件上传
- 上线前 review

## 不什么时候用

- 纯无状态文档任务
- 完全离线、无输入边界的小样例说明

## 输入

- 改动范围
- 数据输入点
- 权限和配置上下文

## 输出

- 安全风险列表
- 加固建议
- 必修项和可后续项

## 执行步骤

1. 先找输入边界、权限边界、输出边界。
2. 看有没有默认放开的高风险点。
3. 检查 secrets、鉴权、校验、错误暴露、依赖风险。
4. 区分必须立刻修和可后续优化。
5. 把关键风险回写到 review 或 release 判断里。

## 常见偷懒借口

- “内部系统先不管”
- “先跑通，再补安全”
- “这个接口现在没人会打”

## 红线与危险信号

- 没有输入校验
- secrets 处理含糊
- 权限边界不清
- 错误直接暴露内部细节

## 验证方式

- 风险点有明确列表
- 默认不安全项被收敛
- 合并或上线判断里体现关键风险

## 交接产物

- 安全 review 摘要
- 风险等级
- 必修项列表

## 真实例子

- 新接口 review
- 文件上传或鉴权变更前检查
