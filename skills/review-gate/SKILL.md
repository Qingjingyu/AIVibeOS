# review-gate

## 解决什么问题

把“能不能合、能不能发”变成一个有门槛的判断，而不是只看感觉或进度压力。

## 什么时候用

- 准备合并
- 准备交付
- 准备进入发布环节

## 不什么时候用

- 还在前期探索
- 改动尚未成形

## 输入

- 改动内容
- 测试结果
- review 结果
- 发布风险

## 输出

- go / no-go 判断
- 必修项
- 可后续项

## 执行步骤

1. 先看行为风险，再看结构质量。
2. 再看安全、性能、文档和发布证据。
3. 必修项不满足时，直接不给过。
4. 可后续项可以记录，但不能掩盖阻塞项。
5. 结论必须给清楚，不打模糊球。

## 常见偷懒借口

- “问题不大，先合了再说”
- “这个风险上线后再看”
- “虽然没证据，但应该能过”

## 红线与危险信号

- 阻塞项被包装成建议项
- 合并判断没有依据
- 发布判断没有回滚和验证

## 验证方式

- 有清楚的 go / no-go 结论
- 阻塞项和建议项被分开
- 结论能追溯到证据

## 交接产物

- gate 结论
- 阻塞项
- 风险说明
