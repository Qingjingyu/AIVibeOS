# browser-testing-with-devtools

## 解决什么问题

在真实浏览器里验证 UI、交互、网络和页面状态，防止“代码看着对，页面其实不对”。

## 什么时候用

- 前端页面开发
- Figma 对稿
- UI Bug 定位

## 不什么时候用

- 纯后端任务
- 没有浏览器界面的任务

## 输入

- 页面地址
- 目标交互
- 对稿或验收标准

## 输出

- 页面截图
- 关键状态证据
- 差异说明

## 执行步骤

1. 打开真实页面，不只读代码。
2. 看 DOM、样式、网络、控制台。
3. 做最关键的交互，不乱点。
4. 留下截图和异常证据。
5. 只修最关键差异，再复测。

## 常见偷懒借口

- “我看代码应该没问题”
- “页面大概像了就行”
- “不用留截图，肉眼看过了”

## 红线与危险信号

- 页面没实际打开过
- 没有截图证据
- 交互状态纯空壳

## 验证方式

- 有浏览器截图
- 有控制台 / 网络检查结果
- 与设计或需求的主要差异说得清

## 交接产物

- 页面截图
- 差异列表
- 复测结论

## 真实例子

- 本地页面验收
- Figma 高保真还原对稿
