# 外部参考仓库与用途清单

这份文档先收录当前对 目标项目 有参考价值的外部仓库和站点。

目标不是照搬，而是明确：

- 这个东西主要解决什么问题
- 对 目标项目 哪类问题有帮助
- 后续应优先学习还是按需查阅

## 优先级建议

### P1：优先吸收

- `OpenSpec`
- `superpowers`
- `web-access`

### P2：按模块吸收

- `everything-claude-code`
- `claude-code-skills`
- `skills.sh`
- `gstack`
- `OpenclawStar`

### P3：按场景参考

- `ui-ux-pro-max-skill`
- `agency-agents`
- `oh-my-openagent`

## 1. OpenSpec

- 链接：<https://github.com/Fission-AI/OpenSpec>
- 关键词：Spec-driven development、proposal、design、tasks
- 适合解决：
  - 开发前需求不清
  - AI 直接吃模糊需求开写
  - 缺少 proposal -> spec -> design -> tasks 的文档闭环
- 对 目标项目 的价值：
  - 适合做你们的项目启动规范骨架
  - 适合沉淀为 `AIVibe` 的 spec 流程参考
- 建议用法：
  - 学它的流程，不要机械照搬所有命令
  - 重点吸收 proposal / spec / design / tasks 的分层方式

## 2. oh-my-openagent

- 原链接：<https://github.com/code-yeongyu/oh-my-opencode>
- 当前仓库：<https://github.com/code-yeongyu/oh-my-openagent>
- 关键词：agent harness、orchestration、IDE / TUI agent
- 适合解决：
  - 多 agent 编排
  - agent 工具壳 / 运行框架参考
- 对 目标项目 的价值：
  - 更偏 agent 运行框架层参考
  - 对 `AiCore` 当前阶段不是第一优先
- 建议用法：
  - 后续在研究 agent 工具壳、任务编排框架时再重点看

## 3. everything-claude-code

- 链接：<https://github.com/affaan-m/everything-claude-code>
- 关键词：skills、instincts、memory、security、research-first development
- 适合解决：
  - AI 开发工作流增强
  - 技能、记忆、研究优先流程
  - 团队 agent 使用规范参考
- 对 目标项目 的价值：
  - 对 `AIVibe` 很有帮助
  - 适合吸收它的“研究优先”“持续学习”“技能化沉淀”思路
- 建议用法：
  - 用来优化团队 AI 使用方法
  - 不建议整套原样搬入，优先吸收方法论

## 4. ui-ux-pro-max-skill

- 链接：<https://github.com/nextlevelbuilder/ui-ux-pro-max-skill>
- 关键词：UI/UX、design intelligence、多平台设计技能
- 适合解决：
  - 页面视觉参考不够
  - AI 写前端容易出 demo 感
  - 缺少系统化视觉输入
- 对 目标项目 的价值：
  - 对 `Showa/JB/官网/Nyoro` 这类前台项目有直接帮助
  - 对提升页面质感和前端 spec 有价值
- 建议用法：
  - 作为前端设计和视觉 spec 的辅助参考
  - 不替代真正的产品和设计判断

## 5. claude-code-skills

- 链接：<https://github.com/daymade/claude-code-skills>
- 关键词：skills marketplace、production-ready skills
- 适合解决：
  - 查找现成技能
  - 学习技能组织方式和安装方式
  - 扩展 Claude Code 工作流
- 对 目标项目 的价值：
  - 适合给 `AIVibe` 补“工具清单”和“技能清单”
  - 可作为技能市场参考样本
- 建议用法：
  - 按需找技能
  - 重点关注与 spec、qa、security、review 相关技能

## 6. superpowers

- 链接：<https://github.com/obra/superpowers>
- 关键词：skills framework、software development methodology
- 适合解决：
  - AI 开发流程标准化
  - brainstorming / plan / execution / review 的串联
  - 团队如何把 skill 体系用起来
- 对 目标项目 的价值：
  - 非常高
  - 对你们当前最缺的“规范化开发流程”直接相关
- 建议用法：
  - 重点学习它的方法论和流程切分
  - 回收进 `AIVibe`，变成 目标项目 内部版本

## 7. skills.sh

- 链接：<https://skills.sh>
- 关键词：Agent Skills Directory
- 适合解决：
  - 搜索和发现技能
  - 找同类技能参考
  - 比较技能热度和分布
- 对 目标项目 的价值：
  - 适合作为技能检索入口
  - 不直接提供流程，但对扩展工具库有用
- 建议用法：
  - 当作目录用
  - 用来找适合 spec、ui、qa、research 的技能

## 8. agency-agents

- 链接：<https://github.com/msitarzewski/agency-agents>
- 关键词：specialized agents、agent roles、workflow personas
- 适合解决：
  - 多角色 agent 模型设计
  - 角色职责和工作风格建模
- 对 目标项目 的价值：
  - 对你们未来做多角色 agent 或内部数字员工体系有参考价值
  - 当前不是项目流程第一优先
- 建议用法：
  - 当作角色设计参考库
  - 后续做 agent 组织和岗位化能力时再重点吸收

## 9. web-access

- 链接：<https://github.com/eze-is/web-access>
- 关键词：完整联网能力、三层通道调度、浏览器 CDP、并行分治
- 适合解决：
  - 联网调研
  - 多来源网页读取
  - 浏览器级任务和外网信息抓取
- 对 目标项目 的价值：
  - 很高
  - 对 deep research、竞品分析、外部资料采集、内容调研都很直接
- 建议用法：
  - 可作为后续团队联网调研能力的重要参考
  - 适合放进 `AIVibe` 的研究方法链条里

## 10. gstack

- 链接：<https://github.com/garrytan/gstack>
- 关键词：AI team workflow、plan、review、qa、ship、retro、slash commands
- 适合解决：
  - AI 开发流程角色化不清
  - 计划、评审、QA、发布没有固定动作
  - 团队已经有 spec，但执行层还容易散
- 对 目标项目 的价值：
  - 对 `AIVibe` 很有参考价值
  - 适合作为 AI 团队协作流程和执行节奏的样板
  - 不适合作为 `AiCore` 这种业务底座的技术选型
- 建议用法：
  - 学它的流程编排思想，不照搬命令和目录
  - 重点吸收 `plan / review / qa / retro` 这些执行环节
  - 回收成 目标项目 自己的内部流程规范

## 11. OpenclawStar

- 链接：<https://github.com/Qingjingyu/OpenclawStar>
- 关键词：Figma 设计还原、pixel replica、视觉 diff、页面验收闭环
- 适合解决：
  - Figma 页面还原靠感觉，缺少证据
  - 页面看着像，但素材、参数、状态不完整
  - 设计还原任务经常被做成“顺手重设计”
- 对 目标项目 的价值：
  - 对 `AIVibe` 的 Figma 还原规范有直接价值
  - 特别适合补“从 Figma 到代码再到验收”的最后一公里
  - 对关键页面的高保真还原、截图对稿、状态验证非常有帮助
- 建议用法：
  - 不要整仓照搬
  - 优先吸收 `figma-implement-design` 和 `figma-pixel-replica`
  - 把它和本机已有的 `figma-to-code`、`figma-screen-to-code-with-original-assets`、`figma-mcp-analysis` 组合使用
  - 研究结论见：`11_OpenclawStar_Figma高保真还原学习结论.md`

## 当前总判断

如果从 目标项目 当前最紧迫的问题出发，这批资料里最值得优先吸收的是：

1. `OpenSpec`：补需求与文档闭环
2. `superpowers`：补 AI 开发流程方法
3. `web-access`：补研究与联网能力
4. `everything-claude-code`：补团队 AI 使用体系
5. `gstack`：补团队执行层协作和评审节奏
6. `OpenclawStar`：补 Figma 高保真还原和截图验收闭环

前端质感问题则优先参考：

- `ui-ux-pro-max-skill`

技能生态和可扩展能力参考：

- `claude-code-skills`
- `skills.sh`

多 agent 角色化设计参考：

- `agency-agents`
- `oh-my-openagent`
