# 打包分享说明

这份说明是给“要把 `Figmatocode1.0` 发给别人”的人看的。

目标很简单：

- 不漏文件
- 不漏上下文
- 不让别人拿到之后还得回来追问

## 一、最小分享范围

如果你要把这套能力包完整发给别人，最少要包含整个 `Figmatocode1.0` 文件夹。

至少包含：

- `README.md`
- `CHANGELOG.md`
- `SKILL.md`
- `core/`
- `templates/`
- `examples/`

不要只发其中一两个文件。

因为这套东西不是单篇文档，
而是一整套工作方式。

## 二、推荐分享方式

### 方式 1：直接发整个文件夹

适合：

- 团队内部同事
- 同一台机器或同一仓库环境

### 方式 2：压缩成 zip 再发

适合：

- 发给外部协作者
- 发给不在同一仓库的人

推荐压缩包名：

- `Figmatocode1.0.zip`
- `Figmatocode1.0_YYYY-MM-DD.zip`

### 方式 3：复制到其他项目的 `Top能力/` 下

适合：

- 其他项目也想直接复用这套打法
- 团队做能力沉淀和横向复用

推荐落点：

- `Top能力/Figmatocode1.0/`

## 三、发出去时要同时说清楚什么

至少补这几句话：

1. 这是“高保真还原工作包”，不是自由设计包。
2. 默认原则是原素材、原内容、原参数、原结构优先。
3. 先看 `README.md`，再看 `core/`，需要给 Agent 干活时用 `SKILL.md`。
4. 复杂页默认先拆层，必要时先做关键层参数表。

## 四、给同事时的最短说明

可以直接复制这段：

```md
这是我们整理好的 Figmatocode1.0 正式工作包。

你先看 README，再看 core 里的主流程。
如果你要直接让 Agent 干活，就用 SKILL.md 和 Prompt 模板。

这套东西的原则不是“看图猜一个差不多的页面”，
而是：原素材、原内容、原参数、原结构优先，
并且要保留浏览器截图级验收证据。
```

## 五、分享前检查清单

- [ ] `README.md` 能单独读懂
- [ ] `SKILL.md` 能单独给 Agent 用
- [ ] `Prompt 模板` 能直接复制
- [ ] `交接模板` 能直接复制
- [ ] `验收清单` 能直接勾选
- [ ] `examples/` 里有至少一个复杂页案例
- [ ] 没有明显泄露不该共享的内容

## 一句话总结

**打包分享不是把文件扔出去，而是让别人拿到 `Figmatocode1.0` 就能直接开始干活。**
