# Agent Prompt 模板

这份模板是给人直接复制给 Agent 用的。

目标很简单：

让不同人发出的 `Figma to Code` 任务，输入尽量一致，减少自由发挥。

## 模板 1：页面高保真还原

```md
请按 Figmatocode 1.0 的正式工作流实现这个页面。

任务要求：
- 这是还原任务，不是重设计任务
- 优先使用原素材、原内容、原参数、原结构
- 不允许自己替换图标、文案、封面、比例
- 不允许把整页截图直接糊成背景图交差
- 必须先拉 design context 和 screenshot，再开始实现
- 必须保留浏览器截图级验收证据
- 如果页面属于复杂页，必须先拆层，必要时先做关键层参数表

输入信息：
- Figma URL：<在这里填>
- file key：<在这里填>
- node id：<在这里填>
- 目标页面路由：<在这里填>
- 技术栈：<在这里填>
- 样式体系：<在这里填>

执行要求：
1. 先判断页面类型和复杂度
2. 先分析页面结构和分层
3. 下载并使用原始素材
4. 先搭结构，再接素材，再调细节
5. 对照 Figma 截图做至少一轮浏览器收差
6. 最后汇报剩余差距，不要空口说像素级

输出要求：
- 页面代码
- 用到的素材目录
- 当前页面截图
- 与原稿的主要差异说明
- 测试 / 构建结果
```

## 模板 2：只做结构首版

```md
请按 Figmatocode 1.0 的工作流做这个页面的结构首版，暂时不要追求像素级细节。

要求：
- 先拉 design context、screenshot、metadata
- 先判断页面复杂度
- 先拆页面结构
- 先把背景层、内容层、覆盖层搭出来
- 先用原素材，不要乱替换
- 暂时只做结构正确、素材正确、主要视觉方向正确

输入：
- Figma URL：<在这里填>
- file key：<在这里填>
- node id：<在这里填>
- 目标页面路由：<在这里填>
```

## 模板 3：继续精修上一版

```md
请继续精修这个已经做过一版的 Figma 页面。

要求：
- 不要推翻重做
- 不要改任务目标
- 只收最明显的视觉差异
- 每轮只改最关键的 3 到 5 个问题
- 改完后重新截图，再汇报
- 如果连续两轮仍明显不像，请回头补关键层参数，而不是继续盲调
```
