# 交接话术模板

这份模板是给同事、外包、其他 Agent 做交接时直接用的。

目标：

- 不丢上下文
- 不让下一手重新猜
- 不让页面做到一半又跑偏

## 模板 1：从 0 开始交接

```md
这次任务是 Figmatocode 1.0 的高保真还原，不是重设计。

请按这个原则做：
- 优先使用原素材、原内容、原参数、原结构
- 先拉 design context 和 screenshot
- 不允许整页截图糊背景
- 不允许自己替换图标、封面、文案
- 最后要保留浏览器截图级验收证据
- 如果页面属于复杂页，先拆层，必要时先做关键层参数表

输入信息：
- Figma URL：<填这里>
- file key：<填这里>
- node id：<填这里>
- 目标路由：<填这里>
- 技术栈：<填这里>
- 样式体系：<填这里>
```

## 模板 2：做到一半转交

```md
这个页面已经做过一版，不要推翻重做。

当前状态：
- 已完成：<填这里>
- 已接入原素材：<填这里>
- 已完成浏览器截图对稿：<是 / 否>
- 当前主要差异：<填这里>
- 是否属于复杂页：<是 / 否>
- 是否已补关键层参数表：<是 / 否>

请继续做的事情：
1. <填这里>
2. <填这里>
3. <填这里>

不要做的事情：
- 不要改任务目标
- 不要把还原任务做成重设计
- 不要替换已经确认正确的原始素材
- 不要在没搞清层级前乱调视觉参数
```

## 模板 3：交给验收人

```md
这个页面已经进入验收阶段。

请按这几个维度检查：
1. 页面结构是否和 Figma 主结构一致
2. 原始素材是否正确
3. 文案是否正确
4. 关键视觉参数是否明显跑偏
5. 当前页面和 Figma 的主要差异还剩哪些
6. 页面是否可运行、可构建

请不要只说“感觉差不多”。
请直接指出最明显的 3 到 5 个问题。
```
