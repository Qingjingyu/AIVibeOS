# 页面验收清单模板

这份清单专门用于 `Figma -> 代码` 页面验收。

每一项只填：

- 通过
- 不通过
- 待确认

## 一、输入是否正确

- [ ] 已确认 Figma URL 正确
- [ ] 已确认 file key 正确
- [ ] 已确认 node id 正确
- [ ] 已确认目标页面路由正确
- [ ] 已使用 design context
- [ ] 已使用 screenshot

## 二、素材是否正确

- [ ] 封面 / 图片使用的是原始素材
- [ ] 图标使用的是原始素材或原始 SVG
- [ ] 没有用相似素材替代
- [ ] 没有把整页截图糊成背景图交差

## 三、结构是否正确

- [ ] 页面主结构和 Figma 一致
- [ ] 页面分层关系正确
- [ ] 背景层 / 内容层 / 覆盖层关系正确
- [ ] 关键模块没有缺失

## 四、视觉是否接近

- [ ] 间距节奏基本一致
- [ ] 圆角基本一致
- [ ] 字号 / 字重 / 行高基本一致
- [ ] 色值基本一致
- [ ] 阴影 / 渐变 / 模糊没有明显跑偏
- [ ] 不存在明显“页面气质变了”的问题

## 五、复杂页额外检查

- [ ] 已识别关键层
- [ ] 已拿到关键层截图
- [ ] 已拿到关键层 design context
- [ ] 必要时已整理关键层参数表
- [ ] 没有在层级不清时直接乱调 blur / shadow / opacity

## 六、内容是否正确

- [ ] 标题正确
- [ ] 副标题正确
- [ ] 正文文案正确
- [ ] 按钮文案正确
- [ ] 没有擅自改文案

## 七、交互和状态是否可用

- [ ] 关键按钮不是空壳
- [ ] 页面没有明显报错
- [ ] 至少具备基本可运行性
- [ ] 如果是关键页面，已覆盖关键状态

## 八、证据是否齐全

- [ ] 已保留当前页面截图
- [ ] 已和 Figma 截图对过
- [ ] 已记录当前最明显的差异点
- [ ] 没有在无证据情况下宣称“像素级”

## 九、工程是否通过

- [ ] 页面能正常运行
- [ ] 构建通过
- [ ] 基本测试通过
- [ ] 没有明显影响交付的错误
