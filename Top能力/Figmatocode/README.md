# Figmatocode 1.0

这是 `Figma -> 代码` 的正式 `1.0` 能力包。

它不是历史草稿堆，也不是方法论试验田。
它的定位很明确：

- 只保留当前仍然正确的工作流
- 只保留真正能减少返工的模板
- 只保留最能说明问题的案例
- 去掉会干扰判断的旧叙事

如果一句话概括这套能力：

**先拿设计真值，再写代码；先看浏览器真值，再说还原完成。**

这套目录本身就是完整包。
只看 `Figmatocode1.0/`，就应该能直接开工、交接、验收和复用。

## 这套 1.0 解决什么问题

它主要解决这几类典型翻车点：

1. 看到 Figma 截图就直接开写，结果结构和素材都猜错。
2. 页面表面像了，但层级、状态、节奏不对。
3. 复杂页只会硬抠 CSS，最后玻璃感、白雾、蒙版全部发假。
4. 验收只靠嘴说“差不多”，没有截图证据。
5. 旧方法、新方法、案例、模板全混在一起，后来的人不知道该先看哪份。

## 当前唯一推荐工作流

`Figma 真值 -> 页面分型 -> 结构拆层 -> 原素材接入 -> 参数收敛 -> 浏览器截图对稿 -> 工程验证`

复杂页额外加一道：

`关键层参数表 -> 分层实现`

如果你只记一条原则，就记这条：

**先把标准答案拿全，再开始实现。**

## 这份 1.0 的边界

这套正式版默认覆盖：

- 页面高保真还原
- 复杂页拆层
- 关键层参数工作流
- Agent 发起方式
- 交接方式
- 验收方式
- 对外打包方式

它不负责的，是“把整个 Figma to Code 做成全自动产品”。
它首先是一套能稳定交付页面的正式工作包。

## 完成标准

在这套 1.0 里，“完成”默认至少意味着：

1. 页面使用了设计真值，不是靠猜。
2. 页面使用了正确素材，不是相似替代。
3. 页面结构和关键视觉方向对得上。
4. 页面在浏览器里看过、截过、对过。
5. 页面不是假按钮或纯空壳。
6. 还有差异就明确写差异，不空口说像素级。

## 一句话总纲

**这不是让 AI 自由发挥的包，这是让团队稳定做高保真还原的正式包。**
