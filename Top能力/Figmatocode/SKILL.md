# Figmatocode 1.0

## 目标

把 Figma 页面尽量高保真地落成代码，并且让结果可验收、可复盘、可交接。

默认假设：

- 你只拿到了这一个 `Figmatocode1.0/` 目录
- 你不需要再额外寻找补充说明

重点不是“先生成一个差不多的页面”，而是：

- 用原素材
- 用原内容
- 用原参数
- 用原结构
- 保留浏览器侧验收证据

## 什么时候必须用

当用户给你：

- Figma URL
- file key
- node id
- 明确页面或组件

并要求：

- 高保真还原
- 尽量 1:1
- 不要自己改设计
- 不要把任务做成重设计

时，必须使用这套 skill。

## 不适用场景

下面这些情况不要用这套 skill：

1. 用户明确要重设计。
2. 没有 Figma，只想快速做一个低保真 demo。
3. 用户要的是视觉探索，而不是设计还原。

## 硬规则

### 规则 1：先拿权威输入

正式实现前，至少拿到：

- `design context`
- `screenshot`
- 原始图片 / SVG

如果是复杂页，再额外判断是否需要先填关键层参数表。

### 规则 2：不允许整页截图糊背景交差

除非任务本身就是快照海报，否则：

- 不允许整页截图充背景
- 不允许拿整张裁图冒充页面实现

### 规则 3：还原任务默认不重设计

如果任务是还原：

- 不擅自改布局
- 不擅自改文案
- 不擅自改图标
- 不擅自换素材

### 规则 4：关键页面不能只是像

至少保证：

- 页面可运行
- 基本交互可用
- 关键状态不是纯空壳

### 规则 5：没有浏览器证据，不准说像素级

至少保留：

- Figma 截图
- 当前页面截图
- 当前主要差异说明

## 标准流程

### Step 0：确认任务是还原，不是重设计

先确认目标是不是高保真还原。
如果是，还原优先级高于个人发挥。

### Step 1：拉 Figma 真值

优先拉：

1. `get_design_context`
2. `get_screenshot`
3. `get_metadata`

目的：

- 看结构
- 看层级
- 看素材
- 看参数

### Step 2：判断页面复杂度

如果页面有：

- 毛玻璃
- 白雾
- 蒙版
- 多层卡片穿插
- 强光效
- 多视觉重心

默认按复杂页处理：

- 先拆层
- 必要时先做关键层参数表

### Step 3：下载并使用原始素材

优先用 Figma 给出的原图、SVG、图标。
禁止随手换成相似素材。

### Step 4：先搭结构，再接素材，再调参数

顺序固定：

1. 结构骨架
2. 模块层级
3. 原始素材
4. 关键参数

### Step 5：获取浏览器真值

用真实浏览器或稳定截图脚本：

- 打开本地页面
- 固定 viewport
- 截图
- 和 Figma 对照

### Step 6：只收最明显差异

每轮只改最关键的 3 到 5 个问题。
不要一轮里乱改十几个地方。

### Step 7：做工程验证

至少确认：

- 页面能运行
- 基本构建通过
- 必要测试通过
