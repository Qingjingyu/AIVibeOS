# AIVibe OS Guardrails 规范

## 目标

定义 AIVibe OS 里什么情况下要拦输入、拦输出、直接中断任务。

## 总原则

Guardrails 不是额外安全层，而是 AI Task 的默认挂点。

每个准备进入生产的 Task，至少要明确：

- 输入 guardrail
- 输出 guardrail
- tripwire 触发后的动作

## 输入 Guardrails

适合拦这些问题：

- 输入为空或结构不合法
- 缺少关键上下文
- 含敏感内容
- 超出业务允许范围

## 输出 Guardrails

适合拦这些问题：

- JSON 结构不合法
- 关键字段缺失
- 内容明显偏题
- 命中敏感内容
- 输出超长或不可消费

## Tripwire 之后怎么办

默认只允许 4 种动作：

1. 直接中断
2. 返回保底结构
3. 转人工
4. 进入降级路径

## 一句话铁规矩

**Guardrail 的价值不是“提醒一下”，而是让不合格输入输出不能悄悄混进生产链路。**
