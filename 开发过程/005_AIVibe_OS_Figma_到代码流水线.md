# AIVibe OS Figma 到代码流水线

## 目标

解决“设计稿每次都重做、页面还原全靠手搓、相似界面反复写”的问题。

原则很明确：Figma 不是直接变页面，而是先变成可复用资产。

## 标准路径

```text
Figma
-> Design Tokens
-> 基础组件
-> 页面模块
-> 业务页面
```

## 第一步：提取 Design Tokens

先从 Figma 里抽出最稳定的东西：

- 颜色
- 字体
- 间距
- 圆角
- 阴影
- 栅格

这一步的目标不是做页面，而是建立统一视觉语言。

## 第二步：沉淀基础组件

把高频元素做成组件，而不是在页面里重复拼：

- Button
- Input
- Select
- Modal
- Card
- Table
- Tabs

每个组件必须先定义：

- 变体
- 状态
- 尺寸
- 禁用和异常表现

## 第三步：组装页面模块

把“搜索栏 + 筛选器 + 列表 + 分页”这类组合沉成页面模块。

这样后面同类页面只需要拼模块，而不是从像素开始还原。

## 第四步：生成业务页面

页面层只做两件事：

- 组装模块
- 接真实数据和真实交互

业务页面不应该重新定义视觉规则，也不应该复制基础组件实现。

## AI 在这条链路里的角色

AI 可以负责：

- 从 Figma 提取 tokens 和组件信息
- 生成基础组件初稿
- 对照设计稿补状态和样式
- 批量生成页面模块骨架

但人必须拍板：

- 哪些是通用组件
- 哪些只是某个页面的特例
- 哪些交互值得沉淀成规范

## 一条铁规矩

**同一类界面元素，只允许还原一次；第二次开始必须复用。**
