# AIVibe OS HITL 中断与恢复规范

## 目标

定义什么情况下任务必须停下来等人拍板，以及拍板后怎么继续。

## 总原则

Human-in-the-loop 不是“群里问一句”，而是系统内可中断、可恢复的一部分。

## 必须中断的场景

- 高风险外部调用
- 删除、覆盖、发布这类不可逆操作
- Guardrail 命中 tripwire
- 多 Agent 交接中出现关键歧义
- AI 输出质量无法自动判断

## 中断时必须保存什么

- task_id / thread_id
- 当前步骤
- 已完成步骤
- 待人工决策的问题
- 决策前的关键上下文

## 恢复时必须确认什么

- 人类给出的结论
- 是否需要修改输入或策略
- 从哪一步继续
- 是否需要跳过已完成步骤

## 一句话铁规矩

**该暂停时就暂停，但暂停不能让状态丢失；恢复时必须从清晰的检查点继续。**
