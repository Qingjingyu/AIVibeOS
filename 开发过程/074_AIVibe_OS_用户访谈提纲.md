# AIVibe OS 用户访谈提纲

## 目标

让用户访谈从“收集意见”变成“发现机会”。

## 访谈前准备

- 这次要验证什么问题：
- 受访者是谁：
- 这次明确不问什么：

## 访谈主线

### 1. 目标

- 你想完成什么事：
- 对你来说最重要的结果是什么：

### 2. 现状

- 你现在怎么做这件事：
- 你现在用什么工具或方法：

### 3. 痛点

- 哪一步最麻烦：
- 哪一步最容易出错：
- 哪一步最浪费时间：

### 4. 替代方案

- 如果没有现有工具，你会怎么解决：
- 你试过哪些替代方案：

### 5. 机会点

- 如果有一个更好的方式，它最该解决哪一步：
- 哪种变化最值得你改变现有习惯：

## 访谈后整理

- 真实目标是什么：
- 主要痛点是什么：
- 当前替代方案是什么：
- 值得继续验证的机会是什么：

## 一句话铁规矩

**访谈不是问用户想要什么功能，而是理解他们真正想完成什么。**
