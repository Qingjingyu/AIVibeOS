# AIVibe OS 新项目最小启动路径

## 这份文档解决什么问题

这份文档只回答一个问题：

**拿到一个新项目后，第一天到底先干什么。**

它不是完整手册，也不是路线图。

它是 AIVibe OS 的“开机键”。

如果你现在不想在很多文档里来回跳，只想知道：

- 先看什么
- 先做什么
- 先产出什么
- 什么情况下不能直接开工

那就先看这份。

---

## 一句话先说

**新项目不要一上来就写代码。**

AIVibe OS 规定的新项目最小启动顺序是：

`Research -> Proposal -> Spec -> Design -> Tasks -> Skeleton -> Build`

少一步，后面都容易返工。

---

## 一、启动前先回答 5 个问题

如果下面 5 个问题答不出来，就不要进入开发。

1. 这次到底要做什么产品，解决什么问题？
2. 这次只做哪个最小版本？
3. 这次的核心页面和主流程是什么？
4. 这次有没有 Figma、AI 能力、发布要求？
5. 这次最怕翻车的点是什么？

这 5 个问题不是形式。

它们决定你后面该调用哪套能力。

---

## 二、第一天的最小启动顺序

按下面顺序走，不要跳。

### 第 1 步：先看总纲

先看这 4 份：

1. [000_Roadmap.md](./000_Roadmap.md)
2. [003_AIVibe_OS_标准开发流程.md](./003_AIVibe_OS_标准开发流程.md)
3. [014_AIVibe_OS_完成定义_DoD.md](./014_AIVibe_OS_完成定义_DoD.md)
4. [082_AIVibe_OS_自我优化_Spec_v1.md](./082_AIVibe_OS_自我优化_Spec_v1.md)

目的只有一个：

知道 AIVibe OS 不是资料库，而是一套有顺序、有边界的工作方式。

### 第 2 步：先定这次只做什么

先产出 `Proposal`。

Proposal 至少要写清楚：

- 给谁做
- 解决什么问题
- 这次只做什么
- 这次明确不做什么

如果这一步含糊，后面所有文档都会漂。

### 第 3 步：把需求写实

然后写 `Spec`。

Spec 至少要写清楚：

- 页面清单
- 主流程
- 必须出现的状态
- 接口口径
- 异常情况
- 验收标准

没有 Spec，不允许开工。

### 第 4 步：再定结构

然后写 `Design`。

这里的 Design 不是指视觉稿，而是：

- 前后端怎么分
- 数据怎么流
- 模块怎么拆
- 外部依赖怎么接
- 失败时怎么降级

这一步的目标是先把骨架做稳。

### 第 5 步：拆任务

然后写 `Tasks`。

任务不要写成大口号，要拆成可以直接执行的小块：

- 谁负责
- 改哪里
- 怎么验证
- 完成标志是什么

### 第 6 步：再建工程骨架

这时才进入项目骨架阶段。

先看：

- [013_AIVibe_OS_标准项目仓模板.md](./013_AIVibe_OS_标准项目仓模板.md)
- [026_AIVibe_OS_Env_模板规范.md](./026_AIVibe_OS_Env_模板规范.md)
- [027_AIVibe_OS_Monorepo_初始化清单.md](./027_AIVibe_OS_Monorepo_初始化清单.md)
- [047_AIVibe_OS_最小README_启动模板.md](./047_AIVibe_OS_最小README_启动模板.md)

先把目录、命令、环境变量、README 这些底座搭起来。

### 第 7 步：最后才开始实现

Build 阶段只做一件事：

按 Tasks 实现，不边写边猜。

---

## 三、第一天最少要产出什么

如果是一个正常的新项目，第一天至少应该拿出下面 6 样东西：

1. 一份 `Proposal`
2. 一份 `Spec`
3. 一份 `Design`
4. 一份 `Tasks`
5. 一份最小目录骨架
6. 一份可以让别人接手的 `README`

如果这些还没有，就不要说“项目已经启动了”。

顶多只能说：

**项目刚开始讨论。**

## 五、什么情况下不能直接开工

出现下面这些情况，先停下来，不要进入写代码阶段。

### 1. 用户是谁都还没说清

这会导致页面、流程、功能边界全部漂移。

### 2. 这次只做什么还没定

这会让 MVP 失控，最后又做大又做乱。

### 3. 核心状态没列全

主状态、空状态、报错状态、加载状态缺任何一个，后面都容易返工。

### 4. 外部依赖还没盘清

数据库、对象存储、模型接口、登录方式、部署位置，这些没盘清，后面就会卡死。

### 5. 只有“想法”，没有文档

这不叫开工条件成熟。

这叫大家脑子里各想各的。
