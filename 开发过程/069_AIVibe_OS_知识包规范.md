# AIVibe OS 知识包规范

## 目标

定义什么样的知识资产，才值得被打包成 AIVibe OS 的“知识包”。

## 什么是知识包

知识包不是一堆散落文档，而是一个可被 AI 稳定调用的最小单元。

一个合格的知识包，至少应包含：

- 简短说明
- 原始知识资产
- 统一索引
- 检索入口
- 使用方式

## 推荐结构

```text
knowledge-pack/
  README.md
  references/
  scripts/
  index.json
```

## 每层职责

- `README.md`
  说明这个知识包解决什么问题、怎么用
- `references/`
  放原始资料
- `index.json`
  做唯一索引源
- `scripts/`
  提供统一检索入口

## 适合做知识包的内容

- 主题型学习资料
- 高价值案例库
- 复盘库
- 行业资料库
- 产品 / 设计 / 架构专题资料

## 不适合做知识包的内容

- 临时草稿
- 没有整理结构的碎笔记
- 只适合一次性看的原始资料堆

## 一句话铁规矩

**知识包的核心不是“资料多”，而是“结构清楚，AI 能稳定检索和引用”。**
