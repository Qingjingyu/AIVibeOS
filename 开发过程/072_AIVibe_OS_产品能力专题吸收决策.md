# AIVibe OS 产品能力专题吸收决策

## 这轮研究要吸收什么

基于当前 Lenny 知识库里关于 PMF、产品策略和实验的高信号内容，AIVibe OS 先吸收下面 4 个决策。

## 决策 1：Proposal 要补“问题与假设”

以后 proposal 不应只回答：

- 为什么做
- 给谁做
- 这次做什么

还要补：

- 我们在验证什么问题
- 这次最关键的假设是什么

## 决策 2：MVP 默认定义成“最小验证单元”

以后 AIVibe OS 里的 MVP，不再默认理解成“小一点的产品”，而理解成：

**最小可验证关键假设的版本**

## 决策 3：验收标准要补“价值验证”

现在的验收标准偏功能和工程。

下一步应补：

- 是否验证了用户问题
- 是否验证了关键假设
- 是否拿到了支持或反对这条方向的证据

## 决策 4：后续要补产品实验模板

下一阶段应新增：

1. `AIVibe OS 产品实验模板`
2. `AIVibe OS PMF 判断清单`
3. `AIVibe OS 用户访谈提纲`

## 一句话原则

**AIVibe OS 不只要会把产品做出来，还要会判断这个产品值不值得做。**
