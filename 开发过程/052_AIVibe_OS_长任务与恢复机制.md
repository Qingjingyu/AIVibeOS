# AIVibe OS 长任务与恢复机制

## 目标

定义长任务、多步骤 AI 流程、多 Agent 协作中断后怎么恢复。

## 总原则

长任务的核心不是执行很久，而是：

- 能暂停
- 能保存状态
- 能继续执行
- 不重复已经完成的步骤

## 最低要求

每个长任务在设计时都必须回答：

1. 状态存在哪里
2. 恢复指针是什么
3. 哪些步骤必须幂等
4. 哪些步骤有副作用
5. 失败后从哪一步继续

## 适用场景

- 多步骤 AI 生成
- 人工审批后继续
- 多 Agent 接力
- 文件处理和批量任务

## 恢复设计原则

- 用稳定的任务 ID / thread ID 标识流程
- 每步完成后保存可恢复状态
- 把副作用步骤包裹成可重放、可跳过的单元
- 中断后优先从最近安全检查点继续

## 和 Agent 系统的关系

多 Agent 协作不是纯文档交接。

当未来 AIVibe OS 进入更强的执行层时：

- Agent handoff 应能保存状态
- 人类拍板应能中断并恢复
- 失败后不需要整条链从头重跑

## 一句话铁规矩

**长任务设计的第一原则不是跑完，而是出事后还能接着跑。**
