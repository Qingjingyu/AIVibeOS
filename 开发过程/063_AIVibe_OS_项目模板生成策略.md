# AIVibe OS 项目模板生成策略

## 目标

定义 AIVibe OS 的项目模板从“文档模板”升级到“可生成脚手架”的路线。

## 三步走

### 第一步：文档模板

现在已经有了：

- 目录骨架
- env 规范
- 命令模板
- README 启动模板

### 第二步：参数化模板

下一阶段要补：

- 项目名
- 应用类型
- 是否包含 AI Runtime
- 是否包含 worker

### 第三步：可生成脚手架

最终目标：

- 一次输入参数
- 自动生成基础仓
- 自动带上 docs、命令、env、最小健康检查

## 验收标准

- 不复制旧仓也能起新项目
- 起出来的仓边界一致
- 初始仓能直接启动最小闭环

## 一句话铁规矩

**模板的终点不是“写得很清楚”，而是“真的能自动生成”。**
